# Plano de Trabalho: Modernização e Interoperabilidade do SISGEN - Resumo Executivo

## Informações Gerais

**Projeto**: Parceria RNP – MMA (BID) para Modernização do SISGEN

**Parceiros**:
- **RNP** - Rede Nacional de Ensino e Pesquisa
- **SiBBr** - Sistema de Informação sobre a Biodiversidade Brasileira
- **MMA** - Ministério do Meio Ambiente (Secretaria Nacional de Bioeconomia)
- **Financiamento**: BID (Banco Interamericano de Desenvolvimento)

**Duração**: 13 meses

**Sistema Alvo**: SISGEN - Sistema Nacional de Gestão do Patrimônio Genético e do Conhecimento Tradicional Associado

## Objetivo do Projeto

Fortalecer a capacidade institucional do Ministério do Meio Ambiente (MMA) para a **gestão eficaz do patrimônio genético e do conhecimento tradicional associado**, visando a **repartição justa e equitativa dos benefícios** derivados de seu uso.

### Fundamentação Legal e Internacional

- **Lei de Acesso e Repartição de Benefícios** (Lei 13.123/2015)
- **Protocolo de Nagoya**
- **Marco Global da Biodiversidade**

### Foco Específico

Modernização do SISGEN por meio de:
- Assessoramento técnico
- Capacitação da equipe do MMA
- **Integração com outras bases de dados nacionais e internacionais de biodiversidade**

## Critérios de Sucesso do Projeto

### 1. Conformidade com Princípios FAIR

**FAIR** = Findability, Accessibility, Interoperability, Reusability

Os dados gerenciados pelo SISGEN devem ser:
- **Findable (Encontráveis)**: facilmente localizáveis
- **Accessible (Acessíveis)**: com permissões apropriadas
- **Interoperable (Interoperáveis)**: permitir integração com outros sistemas
- **Reusable (Reutilizáveis)**: para diversas aplicações com base em padrões internacionais

### 2. Integração Efetiva com Sistemas Existentes

**Indicador-chave**: Capacidade de interoperabilidade e integração dos dados do SisGen com outros sistemas de biodiversidade nacionais e internacionais.

**Sistemas de Integração**:
- **SiBBr** - Sistema de Informação sobre a Biodiversidade Brasileira
- **GBIF** - Global Biodiversity Information Facility (escopo prioritário)
- Outros sistemas de dados de biodiversidade

**Tipos de Dados para Troca**:
- Dados de espécies
- Procedências
- Conhecimentos tradicionais associados

**Objetivo**: Promover uma rede de informações mais rica e colaborativa

### 3. Usabilidade e Adoção pelo Usuário

**Público-Alvo**:
- Equipe do MMA
- Outros stakeholders

**Indicadores**:
- Entrada de dados intuitiva e fácil de usar
- Uso consistente do sistema
- Feedback positivo dos usuários

### 4. Capacitação e Empoderamento da Equipe do MMA

**Objetivo**: Capacidade da equipe de **operar e manter o sistema de forma independente** após a conclusão do projeto de consultoria, especialmente no que se refere aos dados de biodiversidade.

### 5. Entrega de Todos os Produtos no Prazo e Orçamento

Incluindo:
- Ferramentas de integração de dados
- Materiais de treinamento
- Manuais técnicos

- Alinhamento de ações
- Acompanhamento de qualidade
- Entrega e segurança da consultoria

### Atividade 2: Mapeamento de Stakeholders e Processos

**Objetivo**: Análise detalhada do estado atual (as-is) para propor ações focadas na interoperabilidade (to-be)

**Método**:
- Entrevistas e reuniões com stakeholders (SBC e SiBBr)
- Mapeamento de usuários
- Análise de processos do SISGEN
- Análise do estado atual de entrada de dados

**Condicionantes**: Cronograma depende da disponibilidade das agendas dos stakeholders

### Atividade 3: Análise Técnica da Arquitetura de Dados

**Objetivo**: Análise técnica profunda da arquitetura de dados atual e do modelo de dados

**Foco**:
- Identificar pontos críticos
- Propor melhorias para otimizar gestão de dados e relatórios do SISGEN
- **Informações sobre biodiversidade e sua ocorrência nos diferentes biomas e regiões do Brasil**

**Objetivo**: Elaboração minuciosa de um Caderno de Recomendações para organização, padronização e armazenamento de dados

**Tipos de Dados**:
- Dados de espécies
- Procedências
- **Conhecimento tradicional associado**

**Padrões e Alinhamentos**:
- Padrões recomendados pelo **SiBBr**
- Padrões do **GBIF** (Global Biodiversity Information Facility)
- Princípios **FAIR**

### Atividade 5: Instalação e Operação do IPT

**Ferramenta**: IPT - Integrated Publishing Toolkit

- Garantir interoperabilidade de dados entre SISGEN, SiBBr e GBIF

**Objetivo**: Organização e padronização alinhada aos padrões internacionais de dados em biodiversidade

**Padrões Internacionais**:

**Dados a serem padronizados**:

- Dados serão disponibilizados no IPT

- Formato dos dados no SISGEN

## Aspectos Técnicos Relevantes para Arquitetura de Dados

### 1. Padrões Internacionais de Dados

**Darwin Core**:
- Padrão internacional para dados de ocorrência de biodiversidade
- Utilizado globalmente pelo GBIF
- Campos padronizados para registros de espécies

**Plinian Core**:
- Padrão para descrição de espécies
- Informações sobre usos de espécies
- Características taxonômicas e ecológicas

### 2. Ferramenta de Interoperabilidade: IPT

**Integrated Publishing Toolkit (IPT)**:
- Ferramenta desenvolvida pelo GBIF
- Permite publicação de dados de biodiversidade
- Facilita transformação de dados para padrões internacionais
- Gerencia visibilidade e acesso aos dados
- Suporta licenciamento adequado

**Infraestrutura**:
- Instalação na infraestrutura da RNP
- Garantia de alta performance
- Armazenamento seguro
- Acesso administrativo pelo MMA

### 3. Princípios de Governança de Dados

**Princípios FAIR**:
- **F**indable (Encontrável): metadados e dados com identificadores únicos
- **A**ccessible (Acessível): protocolos abertos e com controle de acesso apropriado
- **I**nteroperable (Interoperável): uso de vocabulários e padrões comuns
- **R**eusable (Reutilizável): licenças claras, metadados detalhados, proveniência

**Princípios CARE**:
- **C**ollective Benefit (Benefício Coletivo): para povos indígenas e comunidades locais
- **A**uthority to Control (Autoridade para Controlar): povos têm direitos sobre seus dados
- **R**esponsibility (Responsabilidade): uso responsável dos dados
- **E**thics (Ética): respeito aos direitos, interesses e valores

### 4. Arquitetura de Dados

**Componentes Identificados**:

1. **Modelo de Dados do SISGEN** (atual):
   - Necessita análise técnica profunda

2. **Entrada de Dados**:

   - Padronização conforme DarwinCore/PlinianCore

3. **Armazenamento**:
   - Dados de espécies
   - Procedências
   - Conhecimento tradicional associado
   - Registros de ocorrência por bioma/região

4. **Saída de Dados** (Relatórios):
   - Formatos interoperáveis
   - Conformidade com padrões internacionais

5. **Camada de Interoperabilidade** (IPT):
   - Transformação de dados
   - Publicação com controles de acesso
   - Licenciamento

6. **Integração Externa**:
   - SiBBr
   - GBIF
   - Outras bases de dados de biodiversidade

### 5. Metadados

**Importância dos Metadados**:

**Alinhamento**:

### 6. Gestão de Acesso e Segurança

## Relação com o SISGEN

### O que é o SISGEN

**Sistema Nacional de Gestão do Patrimônio Genético e do Conhecimento Tradicional Associado**

**Finalidade**:
- Cadastro de atividades de acesso ao patrimônio genético
- Cadastro de acesso ao conhecimento tradicional associado
- Registro de remessa de patrimônio genético
- Notificação de produtos acabados
- Gestão da repartição de benefícios

**Base Legal**:
- Lei 13.123/2015 (Lei da Biodiversidade)
- Decreto 8.772/2016
- Protocolo de Nagoya

### Desafios Atuais do SISGEN

Com base no projeto, os desafios identificados incluem:

1. **Interoperabilidade Limitada**:
   - Dados não seguem padrões internacionais
   - Dificuldade de integração com outras bases

2. **Arquitetura de Dados**:
   - Necessidade de análise e otimização
   - Pontos críticos a serem identificados

3. **Entrada de Dados**:
   - Formulários não alinhados com padrões internacionais
   - Necessidade de templates padronizados

4. **Gestão de Relatórios**:
   - Saída de dados necessita padronização
   - Foco em informações sobre biodiversidade por bioma/região

5. **Capacitação Técnica**:
   - Equipe necessita formação em padrões internacionais
   - Governança de dados
   - Princípios FAIR e CARE

### Modernização Proposta

**Não inclui**:
- Evolução do sistema SISGEN (desenvolvimento de novas funcionalidades)
- Configurações em ativos de infraestrutura

**Inclui**:
- Análise e recomendações para arquitetura de dados
- Templates para entrada e saída de dados
- Camada de interoperabilidade via IPT
- Capacitação da equipe
- Manuais e documentação

- Acesso ao patrimônio genético

**Impacto no Projeto**:

- Conformidade obrigatória

**Tratado Internacional** (2010):

## Relevância para Arquitetura de Sistema de Conhecimento Tradicional

### 1. Modelo de Referência para Interoperabilidade

O projeto SISGEN oferece um **modelo prático** de como implementar interoperabilidade entre um sistema nacional de gestão de conhecimento tradicional associado e bases de dados internacionais de biodiversidade.

**Componentes-chave**:
- Sistema fonte (SISGEN)
- Camada de interoperabilidade (IPT)
- Sistemas destino (SiBBr, GBIF)

### 2. Padrões de Dados Aplicáveis

**Darwin Core**:
- Aplicável a qualquer sistema que gerencie registros de ocorrência de espécies
- Campos padronizados facilitam integração

**Plinian Core**:
- Especialmente relevante para conhecimento tradicional sobre usos de espécies
- Estrutura para descrever informações etnobotânicas/etnozoológicas

### 3. Princípios FAIR e CARE

**Combinação Essencial**:
- FAIR: para interoperabilidade técnica
- CARE: para governança ética dos dados de povos e comunidades tradicionais

**Aplicação Prática**:
- Dados encontráveis mas com controle de acesso
- Interoperáveis mas com proteção de conhecimento sensível
- Reutilizáveis mas com repartição de benefícios

### 4. Visibilidade Restrita

**Conceito Importante**:
- Dados podem ser disponibilizados com **visibilidade restrita**
- Conforme especificidades de cada conjunto de dados
- Não é tudo ou nada: granularidade de acesso

**Aplicação**:
- Alguns dados podem ser públicos
- Outros restritos a pesquisadores autorizados
- Conhecimento sensível pode ter acesso totalmente restrito

### 5. Licenciamento Adequado

**Necessidade**:
- Dados acompanhados de licenças de uso adequadas
- Proteção jurídica do conhecimento tradicional
- Clareza sobre condições de uso e repartição de benefícios

### 6. Caderno de Recomendações

**Metodologia Replicável**:
- Análise as-is / to-be
- Identificação de necessidades
- Elaboração de recomendações detalhadas
- Templates para padronização

**Aplicável a**:
- Qualquer sistema que necessite modernização
- Projetos que buscam interoperabilidade

### 7. Capacitação como Componente Essencial

**Lição Importante**:
- Tecnologia sozinha não garante sucesso
- Capacitação da equipe é fundamental
- Deve incluir:
  - Aspectos técnicos (padrões, ferramentas)
  - Aspectos jurídicos (LAI, direitos autorais)
  - Governança de dados (FAIR, CARE)

### 8. Manual de Interoperabilidade

**Documentação Essencial**:
- Diretrizes claras para implementação
- Protocolos de segurança
- Procedimentos de manutenção
- Permite operação independente após projeto

### 9. Gestão por Sprints

**Metodologia Ágil**:
- Reuniões quinzenais
- Acompanhamento de qualidade e entregas
- Permite ajustes durante o projeto

### 10. Forte Dependência de Parceiros

**Risco Identificado**:
- Projeto depende de disponibilidade de stakeholders
- Atrasos podem impactar cronograma
- Importante considerar em planejamento

## Desafios e Considerações

### Desafios Técnicos

1. **Análise da Arquitetura Atual**:
   - Identificação de pontos críticos
   - Compatibilidade com padrões internacionais
   - Possíveis limitações técnicas

2. **Padronização de Dados Existentes**:
   - Volume de dados a serem transformados
   - Qualidade dos dados atuais
   - Mapeamento para DarwinCore/PlinianCore

3. **Conhecimento Tradicional Associado**:
   - Dados sensíveis e complexos
   - Necessidade de proteções especiais
   - Equilíbrio entre compartilhamento e proteção

4. **Integração sem Alterar o Sistema**:
   - Não prevista evolução do SISGEN
   - Interoperabilidade via camada externa (IPT)
   - Limitações dessa abordagem

### Desafios Organizacionais

## Conclusão

O Plano de Trabalho de Modernização do SISGEN representa uma **iniciativa estratégica** para:

1. **Fortalecer a gestão** do patrimônio genético e conhecimento tradicional associado no Brasil

2. **Implementar interoperabilidade** de dados de biodiversidade com padrões internacionais

3. **Garantir conformidade** com legislação nacional (Lei 13.123/2015) e compromissos internacionais (Protocolo de Nagoya)

4. **Aplicar princípios FAIR e CARE** de forma prática e integrada

5. **Criar modelo replicável** de modernização e interoperabilidade para outros sistemas

### Relevância para Arquitetura de Sistema de Conhecimento Tradicional

Este projeto oferece **referências concretas** para:

- **Arquitetura de interoperabilidade**: via IPT e padrões internacionais
- **Governança de dados sensíveis**: visibilidade restrita, licenciamento adequado
- **Padronização**: DarwinCore (ocorrências) e PlinianCore (usos)
- **Princípios éticos**: combinação FAIR + CARE
- **Capacitação**: módulos especializados incluindo etnobiologia
- **Documentação**: manuais e protocolos para operação independente
- **Metodologia**: análise as-is/to-be, recomendações, templates
- **Sustentabilidade**: foco em autonomia da equipe após projeto

### Principais Lições

1. **Interoperabilidade não requer mudança do sistema fonte**: camada externa (IPT) pode fazer a transformação

2. **Padrões internacionais são aplicáveis ao contexto brasileiro**: DarwinCore e PlinianCore funcionam para dados de patrimônio genético e conhecimento tradicional

3. **Capacitação é tão importante quanto tecnologia**: equipe precisa conhecer padrões, governança e ferramentas

4. **Visibilidade restrita é possível e necessária**: dados podem ser interoperáveis sem serem totalmente públicos

5. **Princípios FAIR e CARE são complementares**: técnica e ética precisam andar juntas

6. **Documentação garante sustentabilidade**: manuais e protocolos permitem operação futura independente

7. **Metodologia ágil facilita projetos complexos**: sprints quinzenais para acompanhamento

8. **Dependências externas são risco**: disponibilidade de stakeholders pode impactar cronograma
